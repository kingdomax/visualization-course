Introduction

It is time for d3.js! In this assignment you will learn about how to link multiple visualizations with each other. For this, you will implement two standard visualizations that have been introduced in the lecture with d3.js: The scatter plot matrix and the parallel coordinates. These two separate visualizations will then be connected with brushing & linking, so the interaction technique that reflects a user's selection in one view to all other views.

Implement the scatter plot matrix

In a scatter plot matrix, each attribute is assigned to one column and one row of a matrix such that each cell contains a combination of two attributes that are then depicted by a scatter plot.

For this task, it might be helpful to either use a nested for loop or look at the various array functions d3 provides; especially d3.cross() will be of great help in this case.

Regarding coloration, a look at d3.scaleOrdinal will support you greatly. It solidifies the information shown to you regarding how to apply color to data items in the lab class.

Please, always keep the legibility in mind! So, please, do not forget to add labels to the plot, so that the attribute of each row and column is clear. You can orient yourselves at the scatter plot matrix shown in the Lecture 4: Multi-Attribute Visualization.

Hint: Since the scatter plot matrix shown in Lecture 4 uses the same dataset, you can also verify your implementation with it.

You are of course allowed to look at examples of scatter plot matrixes done in d3 in the web. You are free to take ideas from web examples, but we require you to write your own code.

Implement the parallel coordinates plot

In contrast to scatter plot matrixes, in a parallel coordinates plot each attribute is represented by a single axis parallel to all others and each data item shows as a polyline instead of a point.

To implement the PCP, you will need a storage (for example a map) to hold your scales for each axis. The usage of an JavaScript object is not recommended here, since it is not iterable and, therefore, cannot be used to create the axes with the usual join pattern.

Another useful tool, provided by d3 is the d3.line function, that -- given an array of [x,y] values -- creates a string for an SVG (cor canvas) path, that can then be used inside a SVG path element's d attribute.

Brushing & Linking

To benefit more from both views and better understand how a scatter plot would translate into parallel coordinates and the other way around, linking selections of both visualizations is essential.

The example of a brush within a single scatter plot has already given you some hints as to how a d3.brush can be used. Other examples in various contexts can be found here. Please note, that there is not only d3.brush, but also d3.brushX and d3.brushY which might make more sense in the context of the parallel coordinates plot.
